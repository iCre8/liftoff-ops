# Dashboard Operating System Workbook

_Turning metrics into decisions, actions, and scalable operating rhythms_

Reusable workbook for executive, operations, product, program, impact, sales, and AI dashboards.

Use this workbook to define the audience, decisions, metrics, thresholds, actions, ownership, and governance for any dashboard.

## Core Principle

> A dashboard is not finished when the chart is built. A dashboard is finished when the team knows what action to take.

## Workbook Use

| Workbook Use       | Description                                                                                      |
| ------------------ | ------------------------------------------------------------------------------------------------ |
| Executive planning | Clarify north-star outcomes, board metrics, strategic initiatives, and risk triggers.            |
| Program operations | Track attendance, completion, competency growth, support needs, and intervention timing.         |
| Sales and growth   | Track lead quality, conversion, follow-up, pipeline health, offer performance, and lost reasons. |
| AI operations      | Track automation success, human review, error rates, cost, escalations, and governance.          |

Prepared for repeated use. Duplicate pages as needed for each dashboard.

# Exercise 5: Convert Vanity Metrics to Operating Metrics

## Outcome

Turn impressive-looking numbers into metrics that help people decide what to do.

| Vanity Metric   | Why It Is Weak Alone                      | Better Operating Metric                 |
| --------------- | ----------------------------------------- | --------------------------------------- |
| Total users     | Does not show who is active or successful | Active users by cohort or segment       |
| Website visits  | Does not show intent or conversion        | Visit-to-lead conversion rate by source |
| Total learners  | Does not show progress or risk            | Learners on pace vs. behind pace        |
| AI prompts used | Does not show value or reliability        | Successful AI task completion rate      |
| Revenue         | Can hide margin or segment quality        | Revenue by offer, margin, and retention |

| Current Metric | Problem With It | Decision-Ready Replacement |
| -------------- | --------------- | -------------------------- |

|                |                 |                            |

# Exercise 20: AI-Native Dashboard Supplement

## Outcome

Extend the dashboard from descriptive reporting to predictive insight, recommended actions, and agentic workflows.

| Dashboard Mode | What It Answers                | Example                                                   |
| -------------- | ------------------------------ | --------------------------------------------------------- |
| Descriptive    | What happened?                 | Attendance dropped by 8% this week.                       |
| Diagnostic     | Why did it happen?             | Most absences came from one cohort and two specific days. |
| Predictive     | What is likely to happen?      | Five learners are at risk of missing completion targets.  |
| Prescriptive   | What should we do?             | Schedule outreach and assign support sessions.            |
| Agentic        | What can the system safely do? | Draft outreach messages and create follow-up tasks.       |
